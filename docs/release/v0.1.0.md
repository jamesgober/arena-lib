# Release v0.1.0 — Initial Scaffold

**Date:** 2026-05-18
**Compare:** [`v0.1.0`](https://github.com/jamesgober/arena-lib/releases/tag/v0.1.0) — initial release

## Summary

The first published cut of `arena-lib`. Repository created, REPS-aligned standards in place, cross-platform CI configured, and the public surface seeded with a single `VERSION` constant. No allocator API yet — that work begins in `0.2.0`. This release exists to lock the crate name, license, and toolchain expectations before any real implementation lands.

## Added

- Repository layout: `src/`, `tests/`, `docs/`, `.github/`
- `README.md` with project description, badges, and quick-start
- `CHANGELOG.md` in Keep a Changelog format
- `LICENSE-APACHE` and `LICENSE-MIT` (dual license, Apache-2.0 OR MIT)
- `REPS.md` standards placeholder
- `.gitignore`, `.editorconfig`, `rustfmt.toml`, `clippy.toml`
- `Cargo.toml` with MSRV 1.85, edition 2024
- `src/lib.rs` exposing `pub const VERSION: &str` with the full REPS lint posture
- `tests/smoke.rs` verifying `VERSION` is populated
- `docs/API.md` API reference scaffold
- `.github/workflows/ci.yml` — cross-platform CI on Linux, macOS, Windows; stable and MSRV toolchains

## Compatibility

- **MSRV:** Rust `1.85`
- **Edition:** `2024`
- **Platforms:** Linux, macOS, Windows
- **`no_std`:** supported via `default-features = false`

## Next

Phase `0.2.0` — Foundation. Define the public allocator surface: typed arenas, generational indices, error type, and end-to-end smoke tests.
