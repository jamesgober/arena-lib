# Release v0.5.0 — Implementation

**Date:** 2026-05-21
**Compare:** [`v0.2.0...v0.5.0`](https://github.com/jamesgober/arena-lib/compare/v0.2.0...v0.5.0)

## Summary

`arena-lib` `0.5.0` ships the **Implementation** milestone. The 0.2 foundation API stays put — every type, method, and signature is preserved — but the engine room has been rebuilt for production cost models. The interner is now hash-backed for expected O(1) intern and resolve. The bump arena is multi-chunk: `alloc` is effectively infallible, and `reset` retains chunks so subsequent allocations reuse already-mapped memory. A new `DropArena<T>` provides the same alloc-from-`&self` ergonomics for payloads that own resources. Property tests cover the arena, interner, and bump invariants under randomised operation sequences; Criterion benchmarks for every primitive land under `benches/`.

The 0.9 milestone now takes over for the pre-1.0 hardening pass.

## Added

- **`DropArena<T>`** (`arena_lib::drop_arena`) — typed bump-style arena that runs destructors.
  - `DropArena::new()` and `DropArena::with_chunk_capacity(n)` constructors.
  - `alloc(&self, value) -> &mut T` with the same shared-borrow ergonomics as `Bump`.
  - `len`, `is_empty`, `chunk_count` for inspection.
  - Multi-chunk internally: each chunk is a `Vec<T>` filled by direct writes into its spare capacity; outer `Vec<Vec<T>>` reallocations move inner `Vec` headers but not their heap buffers, so handed-out references remain valid.
  - `Send` when `T: Send`; never `Sync`.
  - Destructors honoured via `Drop` only (no in-place reset — that would require dropping live values while outstanding `&mut T` borrows still exist).
- **`Bump::chunk_count`** accessor reporting the number of chunks currently held.
- **Property tests** (`tests/properties.rs`, proptest):
  - Arena: every inserted value is reachable; removed handles never resolve again even after slot reuse; `len` matches `iter().count()` under mixed insert / remove sequences.
  - Interner: idempotent intern; `resolve(intern(s)) == Some(s)`; `lookup` never mutates.
  - Bump: alloc-then-read round-trips for arbitrary value sequences; `allocated_bytes` is monotonic across alloc sequences.
- **Criterion benchmarks** (`benches/`):
  - `arena.rs` — `Arena::insert` (n=16/256/4096), `Arena::get` over 1000 live handles, 1000-element insert+remove churn.
  - `intern.rs` — `intern` over 1000 unique strings, `intern` over 1000 repeated lookups of 10 distinct strings, `resolve` over 1000 hits.
  - `bump.rs` — `alloc<u64>` at n=16/1024/65536, `alloc<[u8; 64]>` x1000, alloc + reset + alloc reuse pattern, `DropArena::alloc<String>` x1000.

## Changed

- **Interner: `BTreeMap` → `hashbrown::HashMap`.** Lookup, intern, and contains are now expected O(1). `Interner::new()` and `Interner::with_capacity` are no longer `const` — `hashbrown::HashMap::new` is not `const`. All other signatures and semantics are preserved.
- **Bump: single-chunk → multi-chunk linked list.** `alloc` and `try_alloc` now grow on demand by allocating a new chunk; failures only surface if the global allocator itself fails. `Bump::with_capacity(n)` pre-allocates an initial chunk of `n` bytes; subsequent chunks default to `max(n, 4 KiB)`. After `reset`, chunks are retained and refilled (chunk 0 first, then chunk 1, …) before any new chunk is requested.
- **`Bump::chunk_capacity()` semantics.** Previously: bytes of the single chunk. Now: total bytes across every chunk currently held. The pre-1.0 minor bump permits this — callers depending on the old "single chunk only" interpretation should use `chunk_count` to detect multi-chunk state explicitly.

## Compatibility

- **MSRV:** Rust `1.85` (unchanged).
- **Edition:** `2024` (unchanged).
- **Platforms:** Linux, macOS, Windows (Tier-1 targets).
- **`no_std`:** supported. `hashbrown` is configured with `default-features = false` and the `default-hasher` + `inline-more` features; no `std` requirement is introduced.
- **New dependencies:**
  - `hashbrown 0.15` (runtime, default-features off).
  - `criterion 0.5` (dev only).
  - `proptest 1` (dev only).
- **Semver:** 0.5 is a minor pre-1.0 bump and is permitted to introduce the behaviour shift in `Bump::chunk_capacity` and the loss of `const` on `Interner::new`. No public type, method signature, or error variant was removed.
